# 案例：ChessVerse 中国象棋 AI 对弈平台

## 项目背景

**项目名称**: ChessVerse (弈界)  
**项目类型**: Web 应用 / AI 对弈平台  
**开发周期**: 2025-03 至 2026-03 (持续迭代)  
**技术栈**: Next.js 16 + React 19 + TypeScript + Prisma + SQLite + Socket.io  

### 项目简介

ChessVerse 是一个融合中国传统文化的 AI 对弈平台，支持：
- 人类玩家与 AI Agent 同台竞技
- 基于 A2A 协议标准接入多模型 AI
- 实时对局、观战、社交功能
- 支持豆包、GPT、Claude、通义千问等多种 AI 模型

### 核心挑战

1. **复杂游戏逻辑**: 中国象棋规则复杂，需要完整的走法验证、将军检测、胜负判定
2. **实时通信**: WebSocket 实现实时对局、消息推送、在线状态管理
3. **Agent 系统**: 设计 Agent 注册、绑定、认证、自动对弈的完整流程
4. **多平台集成**: A2A 协议、SecondMe AI、OAuth2 认证集成

```typescript
// Alpha-Beta 剪枝算法
export class ChessEngine {
  private maxDepth: number;
  
  constructor(difficulty: 'beginner' | 'intermediate' | 'advanced') {
    this.maxDepth = {
      beginner: 2,
      intermediate: 3,
      advanced: 4
    }[difficulty];
  }
  
  // 走法生成 + 评估函数
  getBestMove(board: Board, color: PieceColor): Move {
    return this.alphaBeta(board, color, this.maxDepth, -Infinity, Infinity);
  }
}
```

## 关键决策点

### 决策 1：技术栈选择

**选项对比**:

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| Next.js 全栈 | 一体化、SSR | 学习曲线 | ✅ |
| 前后端分离 | 团队独立 | 部署复杂 | ❌ |
| 传统后端 | 成熟稳定 | 开发效率低 | ❌ |

**决策理由**: Next.js 适合快速迭代，且支持 API Routes + WebSocket

---

### 决策 2：AI 策略设计

**三种策略对比**:

| 策略 | 实现难度 | 用户体验 | 成本 | 选择 |
|------|----------|----------|------|------|
| 纯算法 AI | 低 | 中 | 免费 | 基础版本 |
| 纯 LLM AI | 中 | 高 | 高 | 可选功能 |
| 混合策略 | 高 | 最高 | 可控 | ✅ 推荐 |

**决策理由**: 混合策略结合算法可靠性和 LLM 智能性

---

### 决策 3：Agent 通信协议

**选择 A2A 协议而非自定义协议**:

- ✅ 标准化，易于第三方接入
- ✅ 支持 OAuth2 安全认证
- ✅ Agent Card 自动发现
- ❌ 学习成本较高

### 关键经验

#### ✅ 做得好的

1. **模块化设计**: 游戏核心、Agent 系统、社交系统独立开发，降低耦合
2. **渐进式迭代**: 先完成核心功能，再扩展高级特性
3. **文档驱动**: 复杂功能先写设计文档，再实现
4. **多 Agent 协作**: 代码 Agent 负责实现，统筹 Agent 负责规划

#### ⚠️ 需要改进

1. **测试覆盖**: 早期缺乏自动化测试，后期返工较多
2. **移动端适配**: 初期未考虑移动端，后期需要大量调整
3. **错误处理**: 部分边界情况处理不够完善
