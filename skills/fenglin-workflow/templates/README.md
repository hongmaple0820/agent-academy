# 枫林工作流项目模板

本目录包含枫林工作流的三种项目模板，根据项目复杂度选择使用。

---

## 模板列表

### 1. 简单项目模板

**适用场景**:
- 功能单一的任务
- 2小时内可完成
- 配置修改、Bug 修复、简单脚本

**文件**: [simple-project.md](./simple-project.md)

**特点**:
- 🟢 口头规划即可
- 🟢 快速执行
- 🟢 自动沉淀
- 🟢 无需详细文档

**使用方式**:
```bash
# 复制模板
cp simple-project.md my-project.md

# 快速填写关键信息
# 直接执行
```

---

### 2. 中等项目模板

**适用场景**:
- 功能较复杂
- 2小时-2天可完成
- 新功能开发、模块重构、多平台集成

**文件**: [medium-project.md](./medium-project.md)

**特点**:
- 🟡 简要规划
- 🟡 分步执行
- 🟡 及时沉淀
- 🟡 适度文档

**使用方式**:
```bash
# 复制模板
cp medium-project.md my-project.md

# 填写简要规划
# 分阶段执行
# 记录到 daily log
```

---

### 3. 复杂项目模板

**适用场景**:
- 功能复杂
- 涉及多个模块
- 需要2天以上
- 大型系统开发、架构重构

**文件**: [complex-project.md](./complex-project.md)

**特点**:
- 🔴 详细规划
- 🔴 分阶段执行
- 🔴 系统沉淀
- 🔴 完整文档

**使用方式**:
```bash
# 复制模板
cp complex-project.md docs/plans/YYYY-MM-DD-my-project-plan.md

# 详细填写规划
# 分里程碑执行
# 创建项目档案
# 系统知识沉淀
```

## 如何选择模板

### 决策流程

```
开始
  │
  ▼
项目需要多长时间？
  │
  ├── < 2小时 ──▶ 简单项目模板
  │
  ├── 2小时-2天 ──▶ 中等项目模板
  │
  └── > 2天 ──▶ 复杂项目模板
```

### 快速判断

| 问题 | 简单 | 中等 | 复杂 |
|------|------|------|------|
| 涉及模块数量 | 1个 | 2-3个 | 3+个 |
| 需要学习新技术 | 否 | 可能 | 是 |
| 需要架构设计 | 否 | 简要 | 详细 |
| 需要团队协作 | 否 | 可能 | 是 |
| 风险等级 | 低 | 中 | 高 |

## 模板对比

| 维度 | 简单项目 | 中等项目 | 复杂项目 |
|------|----------|----------|----------|
| **规划时间** | 5分钟 | 30分钟 | 2-4小时 |
| **规划形式** | 口头 | 简要文档 | 详细文档 |
| **文档位置** | 无需 | memory/daily/ | docs/plans/ |
| **审查要求** | 自审 | 推荐审查 | 必须审查 |
| **沉淀要求** | 自动 | 及时 | 系统 |
| **项目档案** | 无需 | 推荐 | 必须 |

## 使用建议

### 简单项目

1. **快速判断**: 5分钟内确定是简单项目
2. **口头确认**: 和用户确认执行计划
3. **直接执行**: 不需要等待详细规划
4. **自动记录**: 完成后自动记录到 daily log

### 中等项目

1. **简要规划**: 30分钟内完成规划
2. **模块划分**: 明确各模块优先级
3. **分步执行**: 按模块逐步完成
4. **及时沉淀**: 每天结束记录进展

### 复杂项目

1. **详细规划**: 写完整的规划文档
2. **架构设计**: 画出系统架构图
3. **里程碑管理**: 设定清晰的里程碑
4. **定期同步**: 定期同步项目状态
5. **系统沉淀**: 创建完整的项目档案

## 模板定制

你可以根据团队需求定制模板：

1. **添加检查清单**: 根据团队规范添加检查项
2. **调整审查流程**: 根据项目类型调整审查要求
3. **添加工具链**: 添加自动化工具命令
4. **定制文档格式**: 调整文档结构和格式

---

## 最佳实践

1. **宁简勿繁**: 能用简单不用中等，能用中等不用复杂
2. **快速启动**: 不要让规划成为执行的阻碍
3. **及时记录**: 当天的事当天记录
4. **持续优化**: 根据实际使用情况优化模板
