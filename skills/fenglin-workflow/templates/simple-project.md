# 简单项目模板

> 适用场景：功能单一、2小时内可完成的任务
> 示例：配置修改、Bug 修复、简单脚本

---

## 项目信息

| 项目 | 内容 |
|------|------|
| **项目名称** | (填写项目名) |
| **项目类型** | (脚本/配置/修复) |
| **预计时间** | < 2小时 |
| **复杂度** | 🟢 简单 |

---

## 阶段 1：理解 (5分钟)

### 需求澄清

- [ ] 用户描述的需求是什么？
- [ ] 核心目标是什么？
- [ ] 有什么约束条件？
- [ ] 期望的输出是什么？

### 复杂度评估

- [ ] 是否涉及多个模块？
- [ ] 是否需要学习新技术？
- [ ] 是否有外部依赖？

**评估结果**: 🟢 简单项目 (< 2小时)

---

## 阶段 2：规划 (口头规划)

### 执行计划

```
口头规划：
"这是一个简单的任务，我计划：
1. (步骤 1)
2. (步骤 2)
3. (步骤 3)

预计 (X) 分钟完成，可以吗？"
```

### 验收标准

- [ ] (标准 1)
- [ ] (标准 2)
- [ ] (标准 3)

## 阶段 4：沉淀

### 自动记录

```markdown
## 项目总结

**项目**: (项目名)
**时间**: (X) 分钟
**结果**: ✅ 成功 / ⚠️ 部分成功 / ❌ 失败

**关键决策**:
- (决策 1)

**遇到的问题**:
- (问题 1) → (解决方案)

**可复用内容**:
- (组件/脚本/配置)
```

### 是否创建技能/脚本

- [ ] 是 → 创建到 `skills/(skill-name)/`
- [ ] 否 → 记录到 `memory/daily/YYYY-MM-DD.md`

---

## 检查清单

### 完成前检查

- [ ] 功能是否正常工作？
- [ ] 是否引入新问题？
- [ ] 文档是否更新？

### 完成后检查

- [ ] 是否记录到 daily log？
- [ ] 是否有可复用内容？
- [ ] 是否需要通知相关人员？

---

*简单项目 = 快速理解 + 口头规划 + 直接执行 + 自动沉淀*
