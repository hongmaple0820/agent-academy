# 每日复盘示例

> 使用 `daily-review` 技能进行完整的晚间复盘

## 📋 场景说明

**时间**: 每天晚上 22:00-23:00
**目的**: 总结当天工作，记录经验教训，规划明天任务
**触发**: 说"每日复盘"或"今日总结"

---

## 💬 完整对话示例

### 第一步：启动复盘

```
用户：每日复盘

小熊：好的，让我帮你进行今天的复盘。🐻

首先，回顾一下今天的主要工作：
1. 完成了什么任务？
2. 遇到了什么问题？
3. 学到了什么新东西？

请简单描述一下今天的整体情况。
```

### 第二步：总结工作

```
用户：今天主要完成了 chat-hub 的消息同步功能，修复了 Redis 连接问题，
还学习了 MCP 协议的基础知识。

小熊：很好！让我记录这些内容：

✅ **完成的工作**
- chat-hub 消息同步功能
- 修复 Redis 连接问题
- 学习 MCP 协议基础

📝 **需要深入的问题**
- Redis 连接问题的根因是什么？
- MCP 协议具体学到了什么？

请详细说明一下 Redis 连接问题的解决过程，这可能会成为宝贵的经验。
```

### 第三步：记录经验

```
用户：Redis 连接是因为订阅的频道不对，应该同时监听 chat:messages 和 
chat:replies 两个频道。

小熊：📌 **经验教训**

**问题**: Redis Pub/Sub 消息丢失
**原因**: 只监听了一个频道
**解决**: 同时监听 chat:messages 和 chat:replies
**分类**: #Redis #PubSub #BugFix

这个经验已经记录到 `memory/core/lessons.md` 中了。

接下来，有什么遗憾或可以改进的地方吗？
```

### 第四步：规划明天

```
用户：明天要完成 MCP 服务器的搭建，还有钉钉插件的优化。

小熊：📅 **明日计划**

1. **MCP 服务器搭建** (优先级：高)
   - 阅读 mcp-builder 技能文档
   - 创建基础项目结构
   - 实现第一个 Tool

2. **钉钉插件优化** (优先级：中)
   - 优化消息格式处理
   - 添加错误重试机制

需要我帮你创建详细的任务计划吗？用 `planning-with-files` 技能。
```

### 第五步：完成复盘

```
用户：好的，创建计划吧。

小熊：✅ 已创建任务计划文件：
- `memory/daily/2026-03-14-plan.md`

📊 **今日复盘完成**
- 工作记录：已存入 `memory/daily/2026-03-14.md`
- 经验教训：已更新 `memory/core/lessons.md`
- 明日计划：已创建任务文件

晚安！明天继续加油！🌙
```

## 💡 使用技巧

### 技巧 1: 固定时间复盘
- 设置每天 22:30 的提醒
- 形成习惯，不容易忘记

### 技巧 2: 诚实记录
- 完成了就是完成了
- 没完成的不要写"完成"
- 参考 `HONEST_VERIFICATION_GUIDELINES.md`

### 技巧 3: 分类标签
- 用 `#Redis` `#BugFix` 等标签
- 方便后续搜索和回顾

### 技巧 4: 关联技能
- 复盘后自动规划明天
- 复杂任务用 `planning-with-files`
- 技术问题用 `systematic-debugging`
