# Redis 配置指南

## 1. 概述

Redis 是一个高性能的键值存储数据库，支持多种数据结构。
本文档介绍如何在生产环境中配置 Redis。

## 2. 安装步骤

### Ubuntu/Debian
```bash
sudo apt update
sudo apt install redis-server
```

### CentOS/RHEL
```bash
sudo yum install redis
```

## 3. 配置说明

### 基本配置
编辑 `/etc/redis/redis.conf`:

```conf
# 绑定地址
bind 127.0.0.1

# 端口
port 6379

# 密码认证
requirepass YourStrongPassword123

# 最大内存
maxmemory 2gb

# 内存淘汰策略
maxmemory-policy allkeys-lru
```

### 持久化配置
```conf
# RDB 快照
save 900 1
save 300 10
save 60 10000

# AOF 日志
appendonly yes
appendfsync everysec
```

## 4. 常见问题

### Q: Redis 连接失败？
A: 检查防火墙和 bind 配置

### Q: 内存不足？
A: 调整 maxmemory 和淘汰策略

- ✅ 启用密码认证
- ✅ 配置持久化
- ✅ 设置内存限制
- ✅ 定期备份
- ✅ 监控性能指标

## 🔧 写作技巧

### 技巧 1: 明确写作目的
- 技术文档 → 准确、清晰、结构化
- 周报 → 简洁、数据化、结果导向
- 邮件 → 礼貌、专业、行动明确
- 文章 → 生动、有吸引力

### 技巧 2: 使用模板
```markdown
# 技术文档模板
1. 概述
2. 安装/配置
3. 使用方法
4. API 参考
5. 常见问题
6. 最佳实践
```

### 技巧 3: 数据支撑
```
❌ "性能提升了"
✅ "响应时间从 500ms 降低到 100ms，提升 80%"

❌ "修复了很多 bug"
✅ "修复了 12 个 Bug，测试覆盖率从 70% 提升到 85%"
```

### 技巧 4: 结构化表达
```
使用金字塔原理：
- 结论先行
- 以上统下
- 归类分组
- 逻辑递进
```
