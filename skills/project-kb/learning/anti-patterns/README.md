# 错误模式 (Anti-patterns)

> 记录项目中遇到的坑和错误模式，避免重复踩坑。

---

## 📝 使用说明

每个错误模式应包含：
- **问题描述** - 什么情况下会出现
- **错误示例** - 代码示例
- **影响** - 会造成什么问题
- **解决方案** - 如何修复
- **预防措施** - 如何避免

---

## 📂 目录

按技术栈分类：

- `javascript/` - JavaScript/TypeScript 常见错误
- `python/` - Python 常见错误
- `go/` - Go 常见错误
- `database/` - 数据库相关错误
- `architecture/` - 架构设计错误
- `deployment/` - 部署运维错误

---

## 🏷️ 标签

| 标签 | 含义 | 颜色 |
|------|------|------|
| `critical` | 严重问题，必须避免 | 🔴 |
| `warning` | 警告，需要注意 | 🟡 |
| `performance` | 性能问题 | 🔵 |
| `security` | 安全问题 | 🟣 |
