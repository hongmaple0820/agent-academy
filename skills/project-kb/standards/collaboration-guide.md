# 协作指南

> 团队协作规范，确保高效沟通和代码质量。

---

## 🤝 协作原则

### 核心原则

1. **尊重** - 尊重他人的时间和贡献
2. **透明** - 工作进展和问题及时同步
3. **质量** - 提交的代码必须经过自检
4. **反馈** - 积极给予和接受建设性反馈

### 沟通规范

| 场景 | 方式 | 时效 |
|------|------|------|
| 紧急问题 | 即时通讯 | 立即响应 |
| 代码审查 | PR 评论 | 24 小时内 |
| 技术讨论 | 文档/会议 | 按需安排 |
| 进度同步 | 日报/周报 | 固定时间 |

---

## 📋 任务协作

### 任务分配

1. **创建任务** - 明确描述、验收标准、截止日期
2. **认领任务** - 主动认领或协商分配
3. **更新状态** - 进行中/阻塞/完成
4. **验收关闭** - 完成后请求验收

### 任务状态

| 状态 | 说明 | 操作 |
|------|------|------|
| 待办 | 等待处理 | 认领任务 |
| 进行中 | 正在开发 | 每日更新进度 |
| 阻塞 | 遇到障碍 | 标记阻塞原因 |
| 审查中 | 等待代码审查 | 提醒审查者 |
| 已完成 | 已合并到主分支 | 关闭任务 |

---

## 💬 代码审查文化

### 审查心态

**作为审查者：**
- 目标是改进代码，不是批评人
- 提问比指责更有效
- 赞赏好的实现

**作为作者：**
- 审查是帮助，不是攻击
- 保持开放心态
- 解释设计决策

### 审查礼仪

#### 给审查者的建议

```
✅ 这样做：
- "这里是否考虑了并发情况？"
- "建议提取为常量，提高可维护性"
- "这个实现很优雅！"

❌ 避免这样：
- "这代码写得不行"
- "你怎么会这样写"
- "明显有 bug"
```

#### 给作者的建议

```
✅ 这样做：
- "感谢反馈，已修改"
- "这里我考虑的是...，你觉得如何？"
- "这个建议很好，已采纳"

❌ 避免这样：
- "你不懂这个设计"
- "不想改"
- 不回复直接修改
```

## 🆘 求助与支援

### 何时求助

- 卡住超过 **30 分钟** 无法解决
- 需要 **架构决策** 建议
- 发现 **潜在风险** 需要评估
- 需要 **跨团队协调**

### 求助方式

1. **描述问题** - 说明尝试过的方案
2. **提供上下文** - 相关代码、错误信息
3. **明确需求** - 需要什么帮助
4. **及时反馈** - 解决后更新状态

### 求助模板

```markdown
【问题】简要描述

【背景】
- 正在实现的功能
- 相关代码位置

【已尝试】
1. 方案 A - 结果
2. 方案 B - 结果

【需要】具体需要什么帮助
```

## 📚 知识分享

### 分享形式

| 形式 | 内容 | 频率 |
|------|------|------|
| 技术分享会 | 新技术/工具 | 双周 |
| Code Review | 代码走读 | 按需 |
| 文档分享 | 经验总结 | 持续 |

### 文档贡献

- 遇到坑 → 更新 `learning/anti-patterns/`
- 好方法 → 更新 `learning/best-practices/`
- 重要决策 → 更新 `learning/decisions/`
- 项目经验 → 更新项目画像

## ⏰ 响应时效

### 审查响应

| 类型 | 时效 | 说明 |
|------|------|------|
| 紧急修复 | 2 小时内 | 生产问题 |
| 普通 PR | 24 小时内 | 日常开发 |

### 问题响应

| 类型 | 时效 | 说明 |
|------|------|------|
| 阻塞问题 | 立即 | 影响开发进度 |
| 技术问题 | 4 小时内 | 需要协助解决 |
| 一般咨询 | 24 小时内 | 非紧急问题 |

## 🎯 冲突解决

### 代码冲突

1. **预防** - 频繁同步主分支
2. **发现** - 提交前检查
3. **解决** - 本地解决后提交
4. **验证** - 确保功能正常

### 意见分歧

1. **表达** - 清晰说明观点
2. **倾听** - 理解对方立场
3. **权衡** - 列出优缺点
4. **决策** - 团队负责人拍板
5. **执行** - 统一执行决策

## 📝 会议规范

### 会议原则

- **必要才开** - 能异步解决的不开会
- **提前准备** - 有议程、有材料
- **准时开始** - 尊重大家时间
- **有结论** - 明确下一步行动

### 会议模板

```markdown
## 会议主题

### 参会人
- 姓名（角色）

### 议程
1. 议题 A
2. 议题 B

### 结论
- 决定 1
- 决定 2

### 行动项
| 任务 | 负责人 | 截止 |
|------|--------|------|
| 任务 A | 姓名 | 日期 |
```
