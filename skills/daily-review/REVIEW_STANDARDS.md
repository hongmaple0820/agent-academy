# 📋 复盘标准规范

> 让每个 AI 都能进行高质量的每日复盘

## 一、复盘的目的

复盘不是流水账，而是**系统性的思考和总结**：

1. **沉淀经验** - 把零散的经历转化为可复用的知识
2. **发现问题** - 识别工作中的痛点和改进空间
3. **规划未来** - 基于过去规划更好的明天
4. **形成记忆** - 建立 AI 的长期知识库

## 二、复盘触发机制

### 自动触发
- **时间**：每天晚上 12:00
- **方式**：systemd timer 自动执行
- **命令**：`systemctl --user start daily-review.service`

### 手动触发
- **关键词**：每日复盘、今日总结、日复盘、开始复盘
- **补复盘**：补昨天的复盘、补 YYYY-MM-DD 的复盘

### 定时任务设置
```bash
# 一键设置
~/.agents/skills/daily-review/scripts/setup-cron.sh

# 检查状态
systemctl --user status daily-review.timer
```

## 三、复盘文档标准格式

```markdown
# 📅 每日复盘 - YYYY-MM-DD

> 补录复盘（如适用）- 补录时间

## ✅ 今日完成

用 checkbox 格式记录完成任务：
- [x] 已完成的任务
- [ ] 未完成的任务

## 🚧 遇到的困难

**格式要求**：
1. **问题标题**
   - 背景：问题的上下文
   - 问题：具体描述
   - 解决方案：如何解决（如已解决）
   - 状态：待解决 / 已解决 / 部分解决

## 📚 学到的知识和经验

**分类记录**：
- **技术知识**：具体的技术点
- **经验教训**：从错误中学到的
- **最佳实践**：验证有效的做法
- **工具技巧**：提高效率的方法

## 📋 计划进度

### 已完成
- [x] 计划项

### 次日计划
- [ ] 明天要做的
- [ ] 要继续跟进的

## 💡 新的想法

**鼓励思考**：
- 新功能想法
- 流程优化建议
- 技术探索方向

## 🎯 方向与目标

### 短期目标（本周）
### 中期目标（本月）
### 长期方向

## 📊 今日数据

用数字说话：
- 工作时长
- 完成任务数
- 代码提交次数
- 文档更新数量

---

*复盘时间：YYYY-MM-DD HH:mm*
*复盘类型：自动 / 手动 / 补录*
```

## 四、复盘信息来源

### 必须读取
1. **今日日志**：`memory/YYYY-MM-DD.md`
2. **Git 提交**：`git log --since="YYYY-MM-DD 00:00" --until="YYYY-MM-DD 23:59"`

### 可选读取
3. **对话历史**：当天的重要对话
4. **任务看板**：项目进度（如适用）
5. **文件变更**：创建/修改的文件列表

## 五、复盘归档规则

### 存储位置
- **复盘文档**：`memory/reviews/YYYY-MM-DD-review.md`
- **今日日志**：`memory/YYYY-MM-DD.md`
- **长期记忆**：`MEMORY.md`

### 归档原则

**什么要归档到 MEMORY.md？**
- ✅ 可复用的经验教训
- ✅ 技术发现和最佳实践
- ✅ 重要决策和原因
- ✅ 配置和环境信息
- ❌ 临时性的问题
- ❌ 未验证的想法

### 归档频率
- 每次复盘后检查是否需要更新 MEMORY.md
- 每周回顾本周复盘，提取重要内容
- 每月总结，精简长期记忆

## 六、多机器人协作规范

### 统一格式
所有 AI（小琳、小猪、小熊）使用相同的复盘格式。

### 独立存储
```
# 小琳
~/.openclaw/workspace/memory/reviews/

# 小猪（Ubuntu 虚拟机）
/home/pig/.openclaw/workspace/memory/reviews/

# 小熊（新电脑）
/home/bear/.openclaw/workspace/memory/reviews/
```

### 共享知识
- 通过 Git 仓库共享有价值的经验
- 重大发现同步到 `ai-chat-room/docs/`
- 定期同步 MEMORY.md 中的通用知识

### 互相学习
- 可以阅读其他 AI 的复盘文档
- 遇到类似问题时参考他人经验
- 避免重复踩坑

## 七、复盘质量标准

### ❌ 不合格的复盘

```markdown
# 每日复盘

今天做了很多事情，很累但很充实。
明天继续努力！
```

**问题**：空洞、无具体内容、无经验总结

### ✅ 合格的复盘

```markdown
# 📅 每日复盘 - 2026-03-11

## ✅ 今日完成

- [x] 完成 OpenClaw 安装脚本 v2.0
- [x] 整理 70+ Skills 分类

## 📚 学到的知识和经验

- **systemd timer**：比 cron 更现代，支持依赖管理
- **技能分类**：10 个大类覆盖完整工作流程

## 📊 今日数据

- Git 提交：5 次
- Skills 整理：70+ 个
```

**优点**：具体、有数据、有经验总结

## 八、复盘检查清单

每次复盘前确认：

- [ ] 读取了今日日志
- [ ] 查看了 Git 提交记录
- [ ] 列出了所有完成任务
- [ ] 记录了遇到的问题
- [ ] 总结了学到的知识
- [ ] 规划了明天的计划
- [ ] 更新了 MEMORY.md（如有重要内容）
- [ ] 文档格式符合标准

## 九、常见问题

**Q: 当天没有完成任务怎么办？**
A: 照常复盘，记录原因和反思，规划改进措施。

**Q: 忘记记录日志怎么办？**
A: 使用补复盘功能，从 Git 记录和对话历史推断。

**Q: 复盘文档太长怎么办？**
A: 保持简洁，重点突出，次要内容折叠或省略。

**Q: 多个项目如何复盘？**
A: 可以按项目分节，或创建项目专属复盘目录。
