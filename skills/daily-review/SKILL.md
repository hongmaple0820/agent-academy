# 每日复盘技能

每天结束时的系统性回顾，帮助你总结经验、规划未来。

## 触发时机

- **自动触发**：每天晚上 12:00（通过 cron 定时任务）
- **手动触发**：用户说"每日复盘"、"今日总结"等关键词
- **补复盘**：用户说"补昨天的复盘"、"补 XX 日复盘"

## 复盘流程

### Step 1: 收集今日信息

从以下来源收集信息：
1. **memory/YYYY-MM-DD.md** - 今日日志
2. **对话历史** - 今天的重要对话
3. **文件变更** - Git 提交记录
4. **任务看板** - 枫林项目进度（如适用）

### Step 2: 生成复盘文档

按以下结构生成 Markdown 文档：

```markdown
# 📅 每日复盘 - YYYY-MM-DD

## ✅ 今日完成

- [ ] 任务1
- [ ] 任务2
- [ ] 任务3

## 🚧 遇到的困难

1. **问题描述**
   - 背景：...
   - 问题：...
   - 解决方案：...（如果已解决）
   - 状态：待解决 / 已解决

## 📚 学到的知识和经验

- 知识点1：...
- 经验教训1：...
- 技术发现：...

## 📋 计划进度

### 已完成
- [x] 计划1
- [x] 计划2

### 明日计划
- [ ] 计划1
- [ ] 计划2
- [ ] 计划3

## 💡 新的想法

- 想法1：...
- 灵感来源：...

## 🎯 方向与目标

### 短期目标（本周）
- ...

### 中期目标（本月）
- ...

### 长期方向
- ...

## 📊 今日数据

- 工作时长：X 小时
- 完成任务：X 个
- 代码提交：X 次
- 文档更新：X 篇

---

*复盘时间：YYYY-MM-DD HH:mm*
```

### Step 3: 归档存储

1. **保存位置**：`memory/reviews/YYYY-MM-DD-review.md`
2. **更新 MEMORY.md**：将重要经验添加到长期记忆
3. **同步到共享知识库**（可选）：`~/.openclaw/ai-chat-room/reviews/`

### Step 4: 提醒机制

复盘完成后：
- 发送到钉钉群（如果用户在线）
- 更新任务看板
- 提醒明日重要事项

## 配置选项

在 `config/local.json` 中可配置：

```json
{
  "dailyReview": {
    "enabled": true,
    "time": "00:00",
    "notify": true,
    "syncToShared": true,
    "updateMEMORY": true
  }
}
```

## 补复盘功能

如果用户说"补昨天的复盘"或"补 XX 日复盘"：

1. 查找指定日期的日志文件
2. 如果日志不存在，从对话历史和 Git 记录推断
3. 生成复盘文档并标注"补录"
4. 更新 MEMORY.md

## 与其他系统集成

- **枫林项目**：自动同步任务看板进度
- **chat-hub**：复盘结果广播到所有 AI
- **MEMORY.md**：重要经验自动归档到长期记忆

## 注意事项

1. 复盘要诚实，既记录成功也记录失败
2. 经验教训要具体，避免空洞的总结
3. 明日计划要可执行，避免过于宏大
4. 数据统计要真实，反映实际工作量
